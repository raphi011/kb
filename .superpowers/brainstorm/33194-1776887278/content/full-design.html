kb — Design Overview

Git-backed markdown knowledge base with web UI, CLI, and git remote

Package Structure

kb/
├── cmd/kb/main.go              # Cobra CLI
├── internal/
│   ├── gitrepo/                # go-git wrapper
│   ├── index/                  # SQLite schema + FTS5 + queries
│   ├── markdown/               # Goldmark: parse frontmatter, links, tags, render HTML
│   ├── kb/                     # KB service (composes gitrepo + index + markdown)
│   └── server/                 # HTTP server + git remote
│       ├── views/              # Templ components (ported from zk-serve)
│       └── static/             # CSS, JS, HTMX, Mermaid
├── go.mod
└── go.sum

Data Flow

Indexing (CLI or post-push)
Git Repo → walk tree / diff → Markdown Parser → title, tags, links → SQLite Index
Full index on first run. Incremental after (diff old_commit..HEAD).
Timestamps from single git log pass. HEAD SHA stored in index_meta.
Server Request Flow
HTTP Request → Handler → KB Service → SQLite + Git (file content)
Metadata from index. File content read from git blobs on demand.
Content negotiation: HTML (browser) / HTMX partial / JSON (API).

SQLite Schema

notes       (path PK, title, body, lead, word_count, created, modified, metadata)
notes_fts   FTS5(title, body, path) content='notes'
tags        (name, path) PK(name, path)
links       (source_path, target_path, title, external) PK(source, target)
index_meta  (key PK, value) — tracks HEAD commit SHA

CLI Commands

kb index [path]              # Full or incremental index
kb search [-i] <query>      # FTS search (-i = fzf interactive)
kb list [-i]                 # List notes (-i = fzf interactive)
kb tags                      # Tags with counts
kb links <path>              # Outgoing links
kb backlinks <path>          # Incoming links
kb cat <path>                # Print raw markdown
kb edit -i                   # fzf select → $EDITOR
kb serve --addr :8080        # Web UI + API + git remote

Server Routes

GET  /notes/{path...}       # Note or folder (HTML / HTMX / JSON)
GET  /search?q=&tags=       # Search results
GET  /tags                  # Tag list
GET  /calendar?year=&month= # Calendar widget
GET  /static/*              # Embedded assets
GET  /healthz               # Health check
GET  /git/...               # Smart HTTP git remote (pull + push)
GET  /                      # → redirect to /notes/

Key Dependencies

go-git Git operations, smart HTTP transport
modernc.org/sqlite Pure-Go SQLite with FTS5
goldmark + extensions Markdown → HTML, wiki-links, frontmatter, syntax highlighting
a-h/templ Type-safe HTML templates
cobra CLI framework
chroma Syntax highlighting for code blocks

Ported from zk-serve

Web UI: 3-column layout (sidebar tree, content, TOC/backlinks), Templ components, HTMX partials, command palette (Cmd+K), tag filtering, calendar widget, dark/light theme, zen mode, keyboard shortcuts, Mermaid diagrams. Adapted to read from KB service instead of zk's notebook.db.

Ported from knowhow

Markdown parser: frontmatter extraction, wiki-link detection, inline #tag extraction, section/heading parsing. Stripped of embedding chunks, query blocks, @mentions.
