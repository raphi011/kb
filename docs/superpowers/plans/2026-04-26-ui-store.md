# UI Store Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Consolidate all UI layout preferences into a single `zk-ui` localStorage key, and add generic panel collapse persistence via `data-panel` attributes.

**Architecture:** New `ui-store.js` module provides `get`/`set` for a single JSON localStorage key. New `panels.js` module uses `data-panel` attributes and a delegated `toggle` listener for generic panel state persistence. Existing modules (`theme.js`, `zen.js`, `resize.js`) migrate to use the store. The inline `<head>` FOUC-prevention script reads the single key.

**Tech Stack:** Vanilla JS (ES modules), esbuild bundling, templ templates, HTMX

**Spec:** `docs/superpowers/specs/2026-04-26-ui-store-design.md`

### Task 1: Create `ui-store.js`

**Files:**
- Create: `internal/server/static/js/ui-store.js`

- [ ] **Step 1: Create the store module**

```js
const STORAGE_KEY = 'zk-ui';
const defaults = { theme: 'dark', zen: false, sidebarWidth: null, tocPanelWidth: null };
let state = null;

function load() {
  try { state = JSON.parse(localStorage.getItem(STORAGE_KEY)) || {}; }
  catch { state = {}; }
}

export function get(key) {
  if (!state) load();
  return state[key] ?? defaults[key] ?? null;
}

export function set(key, value) {
  if (!state) load();
  state[key] = value;
  try { localStorage.setItem(STORAGE_KEY, JSON.stringify(state)); }
  catch {}
}
```

import { set } from './ui-store.js';

export function initZen() {
  const btn = document.getElementById('zen-toggle');
  if (!btn) return;

  function toggle() {
    const active = document.body.classList.toggle('zen');
    document.documentElement.classList.toggle('zen', active);
    set('zen', active);

  btn.addEventListener('click', toggle);

  document.addEventListener('keydown', (e) => {
    if (e.key === 'z' && !e.ctrlKey && !e.metaKey && !e.altKey) {

      toggle();
    }
  });
}

### Task 7: Create `panels.js` and wire it up

**Files:**
- Create: `internal/server/static/js/panels.js`
- Modify: `internal/server/static/js/app.js`
- Modify: `internal/server/static/js/htmx-hooks.js`

- [ ] **Step 1: Create the panels module**

Write to `internal/server/static/js/panels.js`:

```js
import { get, set } from './ui-store.js';

export function restorePanels() {
  for (const el of document.querySelectorAll('details[data-panel]')) {
    if (get('panel.' + el.dataset.panel) === false) {
      el.removeAttribute('open');
    }
  }
}

// Delegated toggle listener — attached once, handles all current and future panels.
document.addEventListener('toggle', (e) => {
  const el = e.target;
  if (el.matches('details[data-panel]')) {
    set('panel.' + el.dataset.panel, el.open);
  }
}, true);
```

Note: The `toggle` event doesn't bubble, so we use `true` (capture phase) for the delegated listener. This is a single global listener that works for all `<details data-panel>` elements including those added by HTMX swaps.

The `restorePanels()` function is separate because it needs to re-run after HTMX swaps replace DOM content with server-rendered `<details open>`.
